# mysql-event-stream v1.3.2 Release Notes

**Release Date:** 2026-05-17
**Type:** Bug Fix
**Previous Version:** v1.3.1

---

## Overview

Version 1.3.2 is a correctness, security, and testing-infrastructure release. Authentication is migrated off deprecated OpenSSL one-shot digest APIs to the modern `EVP_MD_CTX` interface with hardened secure-cleanse coverage. The row decoder now rejects `ROWS_EVENT` payloads whose column count disagrees with the `TABLE_MAP` metadata, and `CdcEngine` propagates row-decode errors through `IsError()` / `ErrorCode()` / `Feed()` consistently. All READMEs document the thread-safety contract, and a `make test-tsan` target now runs the non-E2E suite under ThreadSanitizer.

**Highlights:**

- **OpenSSL EVP auth migration** — `mysql_auth.cpp` no longer uses deprecated `SHA1()`/`SHA256()` one-shots or `SHA_CTX`/`SHA256_CTX`; all intermediate hash buffers are now wiped via `SecureCleanse`
- **Row decoder column-count validation** — Mismatched `ROWS_EVENT` payloads return `MES_ERR_DECODE_ROW` instead of silently decoding garbage
- **CdcEngine error propagation** — Row-decode failures are now visible through `IsError()` regardless of parser state-machine transition; `Feed()` short-circuits on entry and breaks mid-stream
- **Python input validation** — Negative `set_max_queue_size()` values are rejected before crossing the FFI boundary
- **Thread safety documentation** — Five READMEs (EN/JA + Node + Node-npm + Python) describe the single-owner / any-thread-stop contract
- **`make test-tsan`** — Debug+ThreadSanitizer build target for catching data races locally
- **CMake test boilerplate** — `mes_add_gtest()` / `mes_add_e2e_gtest()` helpers replace 17 copies of identical 4-line blocks

No public API or wire-format changes versus v1.3.1. All four E2E matrix targets (MySQL 8.4 / 9.1, MariaDB 10.11 / 11.4) pass for both C++ and Node.js suites.

## Bug Fixes

### 1. OpenSSL EVP Auth Migration

**Type:** Security / Correctness
**Severity:** Medium

`core/src/protocol/mysql_auth.cpp` previously called `SHA1()` and `SHA256()` one-shot functions and used `SHA_CTX` / `SHA256_CTX` low-level structs — both deprecated in modern OpenSSL builds and slated for removal. The implementation now uses `EVP_MD_CTX` via a new `ComputeDigest()` helper that accepts a `DigestPart` array for multi-segment hashing, eliminating Init/Update/Final boilerplate.

`SecureCleanse` RAII coverage was also extended to all intermediate hash buffers, including the previously unprotected `scramble` and `hash3` outputs, so that password-derived material is wiped from the stack on scope exit.

**Files Changed:**

- `core/src/protocol/mysql_auth.cpp` — `ComputeDigest()` helper, EVP digest path, full SecureCleanse coverage
- `core/src/protocol/mysql_auth.h` — Added `evp.h` include alongside `sha.h`

### 2. Row Decoder Column-Count Validation

**Type:** Correctness
**Severity:** High

`ParseRowsContext()` in `core/src/row_decoder.cpp` now validates that the `ROWS_EVENT` column_count matches the `TABLE_MAP` metadata column count, and rejects mismatched events with `MES_ERR_DECODE_ROW` instead of silently decoding garbage bytes as column values. Also removed the unused `AppendZeroPadded()` helper.

**Files Changed:**

- `core/src/row_decoder.cpp` — column-count validation in `ParseRowsContext()`, dead-helper removal
- `core/tests/test_row_decoder.cpp` — `RejectsColumnCountMismatch` unit test

### 3. CdcEngine Error Propagation

**Type:** Correctness
**Severity:** High

`IsError()` previously only inspected the parser state machine, so row-decode failures that recorded `last_error_` but did not cause a parser state transition were invisible to callers. The new semantics:

- `IsError()` returns true when `last_error_ != MES_OK` OR the parser is in `kError`
- `ErrorCode()` returns `last_error_` first and falls back to `MES_ERR_PARSE` only for pure parser failures
- `Feed()` returns `0` immediately if `IsError()` is true at entry, and breaks its inner loop on `IsError()` after each event so consumption stops mid-stream on a decode failure
- The comment on `last_error_` now reflects its engine-level scope

**Files Changed:**

- `core/src/cdc_engine.cpp`, `core/src/cdc_engine.h` — error propagation fixes
- `core/tests/test_cdc_engine.cpp` — `TruncatedRowEventSetsDecodeError` test
- `core/tests/test_capi.cpp` — `FeedReturnsDecodeErrorForTruncatedRowEvent` test, including post-`Reset()` recovery via a valid event stream

### 4. Python Binding Input Validation

**Type:** Correctness
**Severity:** Low

`set_max_queue_size()` now raises `ValueError` for negative values before crossing the FFI boundary, preventing implementation-defined sign-extension when the value is converted to the unsigned C type.

**Files Changed:**

- `bindings/python/src/mysql_event_stream/engine.py`
- `bindings/python/tests/test_engine.py` — `test_negative_max_queue_size_rejected`

### 5. Python Type Annotation Fix

**Type:** Correctness
**Severity:** Low

`_client_configured_libs` was annotated as a string (`"WeakValueDictionary[int, ctypes.CDLL]"`) which prevented static checkers from validating it. The annotation is now bare (no quotes), which is valid under Python 3.11+ without `from __future__ import annotations`.

**Files Changed:**

- `bindings/python/src/mysql_event_stream/_ffi.py`

### 6. Node Addon Unused-Parameter Warnings

**Type:** Build hygiene
**Severity:** Low

`(void)info;` casts were added to `Stop()`, `Disconnect()`, and `Destroy()` N-API callback methods to suppress unused-parameter warnings under `-Wall -Werror`.

**Files Changed:**

- `bindings/node/src/addon/client_wrap.cpp`
- `bindings/node/src/addon/engine_wrap.cpp`

## New Features

### 1. `make test-tsan` Target

The `Makefile` now exposes a `test-tsan` target that configures a Debug build with `MES_ENABLE_TSAN=ON` into `build-tsan/` and runs all non-E2E `ctest` cases under ThreadSanitizer. Useful for catching data races locally before pushing.

**Files Changed:**

- `Makefile`

### 2. Thread Safety Documentation

A "Thread Safety" section was added to all five user-facing READMEs documenting the single-owner contract for `CdcEngine` and the any-thread `stop()` cancellation path for `BinlogClient` / `CdcStream`.

**Files Changed:**

- `README.md`
- `README_ja.md`
- `bindings/node/README.md`
- `bindings/node/README.npm.md`
- `bindings/python/README.md`

## Changed

### 1. CMake Test Helper Functions

Introduced `mes_add_gtest()` and `mes_add_e2e_gtest()` helper functions in `core/tests/CMakeLists.txt` and `core/tests/e2e/CMakeLists.txt`. The helpers replace 17 copies of the same 4-line `add_executable` / `target_link_libraries` / `gtest_discover_tests` boilerplate, reducing the combined file size from ~118 to ~68 lines.

E2E tests now link `GTest::gtest` (not `gtest_main`) and share a `test_e2e_main.cpp` entry point. All `gtest_discover_tests()` calls receive `DISCOVERY_TIMEOUT 30` so slow test binaries do not silently drop tests at configure time.

**Files Changed:**

- `core/tests/CMakeLists.txt`
- `core/tests/e2e/CMakeLists.txt`
- `core/tests/e2e/test_e2e_main.cpp` (new)
- `core/tests/e2e/test_e2e_helpers.h` — `IsE2eServerAvailable()` probe
