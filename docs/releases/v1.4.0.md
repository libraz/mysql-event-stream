# mysql-event-stream v1.4.0 Release Notes

**Release Date:** 2026-06-26
**Type:** Minor (additive API + correctness fixes)
**Previous Version:** v1.3.2

---

## Overview

Version 1.4.0 resolves a commercial-quality, use-case-driven audit of the
engine and both bindings. It fixes defects that silently corrupted data or
crashed on a normal shutdown, lands two consolidation fixes that each resolve a
cluster of findings at their shared root cause, and closes cross-surface API
gaps so the C ABI, Node.js, and Python bindings behave consistently.

All public API changes are **additive** — there are no breaking changes versus
v1.3.2. Three new C ABI functions and one new event field are introduced, and
both bindings gain the structured-log callback and typed error codes.

**Highlights:**

- **TABLE_MAP optional metadata is parsed** — UNSIGNED columns decode correctly
  from raw binlog bytes without a metadata side-connection (single largest root
  cause in the audit)
- **Checksum handling unified** — `binlog_checksum=NONE` streams are no longer
  truncated by 4 bytes per event; the algorithm is auto-detected from the
  `FORMAT_DESCRIPTION_EVENT` and overridable via `mes_set_checksum_enabled()`
- **Python shutdown is crash-safe** — `close()` no longer races an in-flight
  `poll()` into a use-after-free
- **GTID, temporal, and float decoding fixes** — multi-UUID resume, negative
  TIME2/DATETIME2, and big-endian FLOAT/DOUBLE all corrected
- **Cross-surface contracts** — structured log callback, typed errors, and
  metadata-resolution status exposed uniformly across bindings

Verified against all four E2E matrix targets (MySQL 8.4 / 9.1, MariaDB
10.11 / 11.4) for the C++, Node.js, **and** Python suites.

## New Features

### 1. `mes_set_checksum_enabled()` (C ABI)

Explicitly control binlog checksum stripping on a `CdcEngine`. The parser also
auto-detects the checksum algorithm from the `FORMAT_DESCRIPTION_EVENT`, so
`binlog_checksum=NONE` streams (the MariaDB default) are handled correctly out
of the box. The same override is exposed as Node
`CdcEngine.setChecksumEnabled()` and Python
`CdcEngine.set_checksum_enabled()` for raw streams that begin after the FDE.

### 2. `mes_sizeof_event()` / `mes_sizeof_column()` (C ABI)

Report the exact `mes_event_t` / `mes_column_t` struct sizes. The Python binding
now asserts `ctypes.sizeof` against the library-reported sizes, catching ABI
layout drift across pointer widths and struct packing instead of relying on a
loose byte-range heuristic.

### 3. `names_resolved` field on `mes_event_t`

Distinguishes "this table has no column names" from "metadata lookup failed",
plumbed through both bindings so consumers can detect silent metadata
degradation.

### 4. Structured log callback in both bindings

- **Node:** `setLogCallback()` / `LogLevel`, backed by an unreferenced,
  environment-cleaned thread-safe function with a bounded 256-record queue and
  aggregated overflow warnings.
- **Python:** `set_log_callback()` / `LogLevel`.

Exposes the previously C-ABI-only diagnostic callback (decode warnings, auth
traces, truncation notices) to binding users.

### 5. Typed error mapping

Node errors carry a numeric `code` and a category name; the Python client maps
error codes to the same typed exceptions as the engine, so error handling no
longer depends on brittle substring matching.

### 6. libFuzzer harness

`MES_ENABLE_FUZZ` (Clang-only) builds a fuzz target that drives `mes_feed` /
`mes_next_event` with chunked, malformed input through the header parser, state
machine, metadata parser, and row decoder. Intended to run alongside ASan/UBSan.

## Bug Fixes

### 1. Python use-after-free on shutdown

`BinlogClient.close()` serializes against an in-flight `poll()` via a lock, and
`CdcStream.close()` unblocks and awaits the in-flight poll task before teardown.
A normal `break` out of `async for`, `async with` exit, or GC during streaming
no longer crashes.

### 2. UNSIGNED columns decoded as signed

The TABLE_MAP optional metadata block (SIGNEDNESS, COLUMN_NAME) following the
null bitmap is now parsed. SIGNEDNESS is present even in the default MINIMAL
`binlog_row_metadata`, so UNSIGNED columns decode correctly from raw binlog
bytes with no metadata side-connection. An UNSIGNED BIGINT that overflows int64
surfaces as a string instead of a wrong `0`.

### 3. `binlog_checksum=NONE` corruption

Previously the engine unconditionally stripped a 4-byte checksum, silently
truncating every event on NONE streams (corrupting TABLE_MAP trailing metadata
and the last row of ROWS events). The parser now tracks a checksum flag
(auto-detected from the FDE, overridable via `mes_set_checksum_enabled()`).

### 4. Duplicate event replay on multi-UUID GTID resume

`ConvertSingleGtidToRange` now normalizes each comma-separated SID independently
(`uuid:N` → `uuid:1-N`), instead of skipping normalization whenever more than
one UUID is present and re-sending already-processed events.

### 5. Negative TIME2 / DATETIME2 decoding

Negative temporal values are reconstructed from the combined signed packed value
(MySQL complement form) rather than decoding the fractional part independently;
the lossy 10-bit hour mask is dropped.

### 6. FLOAT / DOUBLE on big-endian builds

Decoded through a byte-order-independent little-endian read before
reinterpreting the bit pattern.

### 7. Stale column metadata after DDL

QUERY_EVENT DDL (ALTER/RENAME/DROP/CREATE/TRUNCATE) invalidates the metadata
cache, so column names and signedness are refreshed after a live schema change.

### 8. MariaDB checksum false detection

Checksum state is reset before the detection query, so a failed
`@@global.binlog_checksum` lookup no longer leaves verification stuck enabled
and failing every event with `MES_ERR_CHECKSUM`.

### 9. Engine lifecycle

The TABLE_MAP registry (and derived filter cache) is cleared on ROTATE, and the
recorded resume position rolls back on a decode failure so a reconnect re-reads
the offending event instead of skipping it.

### 10. Protocol boundary safety

verify-CA/identity TLS modes assert the peer certificate and
`SSL_get_verify_result() == X509_V_OK`; `verify_identity` binds IP literals via
`X509_VERIFY_PARAM_set1_ip_asc`, rejects empty hostnames, and omits SNI for IP
literals (RFC 6066). Handshake parsing uses a length-checked fixed-int read;
truncated result-set/row data is a parse error rather than fabricated NULLs;
DEPRECATE_EOF framing follows the negotiated capability; the validator rejects
`binlog_row_value_options=PARTIAL_JSON`; the metadata fetcher scrubs its stored
password on teardown.

### 11. Node binding correctness

Unconsumed `feed()` bytes are retained and prepended to the next poll (no silent
drop under a queue limit); `maxQueueSize` is read as 64-bit and rejects
negatives; auth/validation failures fail fast instead of being retried.

## Changed

- **`mes_set_max_event_size(0)` now means "no limit"** (the absolute cap still
  applies) instead of rejecting every event, matching sibling APIs.
- **Client and parser event-size ceilings are aligned.** C clients can use
  `mes_client_set_max_event_size()` alongside `mes_set_max_event_size()`;
  Node/Python stream configuration exposes one `maxEventSize` /
  `max_event_size` value for both layers. MySQL's one-byte replication OK
  prefix is accounted for separately, so an event exactly at the cap passes.
- **The async client queue now has a byte budget as well as an event-count
  limit.** The default charged-payload budget is 256 MiB and can be configured
  with `mes_client_set_max_queue_bytes()`, `maxQueueBytes`, or
  `max_queue_bytes`; large bursts backpressure the reader before unbounded
  packet memory accumulates.
- **Client lifecycle state is explicit.** `isConnected` / `is_connected`
  reports whether the transport remains usable, while `isStreaming` /
  `is_streaming` reports whether polling can still drain data or one queued
  terminal error. Reader failure, stop, and disconnect no longer leave a dead
  transport reported as connected.
- **Heartbeat timing now follows the read timeout.** The period is
  `min(3 seconds, read_timeout / 2)`, so 1–3 second socket deadlines no longer
  expire before an idle heartbeat. Failure to configure the heartbeat now
  fails stream startup instead of deferring to a reconnect loop.
- **Node row records preserve special column names.** `__proto__`,
  `constructor`, and `prototype` are defined as own enumerable data properties;
  they no longer invoke inherited setters or disappear from the row image.
- **Runtime table-filter changes take effect immediately.** Existing TABLE_MAP
  entries are re-evaluated when include/exclude filters change, without waiting
  for the server to send another TABLE_MAP event.
- **Large binary rows use less transient and retained memory.** BLOB, JSON, and
  GEOMETRY values copy directly into their final storage, and parser buffers
  above 8 MiB are released after each event (all buffers are released on reset).
- Root English/Japanese quick starts now use the exported `CdcEngine`; CI
  compiles and import-smokes both languages' README snippets.
- macOS prebuilt Node packages and Python wheels now require macOS 15.0 on
  both x64 and arm64. macOS is treated as a development platform; the release
  jobs build and clean-install on matching macOS 15 runners instead of claiming
  untested compatibility with macOS 10.15/11.
- **MySQL 8.4 tagged GTIDs (`uuid:tag:N`)** are rejected with a clear log rather
  than an opaque interval-parse failure.
- TABLE_MAP binlog signedness is authoritative; the metadata fetcher no longer
  overwrites it with possibly stale `SHOW COLUMNS` data.

## Migration Guide

No action is required to upgrade from v1.3.2 — the release is backward
compatible. To take advantage of the new behavior:

1. **UNSIGNED columns** now decode correctly with no code change. If you
   previously worked around wrong signed values from the raw-engine path, you
   can remove that workaround.
2. **`binlog_checksum=NONE` (e.g. MariaDB default):** streams now decode without
   truncation automatically. Call `mes_set_checksum_enabled(engine, 0)` only if
   you need to override the auto-detection.
3. **Diagnostics:** register `setLogCallback` (Node) / `set_log_callback`
   (Python) to receive structured decode/auth warnings.
4. **Error handling:** Node errors now expose a numeric `code`; prefer it over
   message-string matching. Python client errors are now the same typed
   exceptions as the engine.
5. **Shutdown (Python):** no change required; closing a client or stream during
   an in-flight `poll()` is now safe.
