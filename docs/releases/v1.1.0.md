# mysql-event-stream v1.1.0 Release Notes

**Release Date:** 2026-04-09
**Type:** Feature
**Previous Version:** v1.0.1

---

## Overview

Version 1.1.0 adds MySQL 9.x compatibility with two major features: RSA public key authentication for `caching_sha2_password` without TLS, and native VECTOR column type support. The project no longer depends on the deprecated `mysql_native_password` plugin.

**Highlights:**

- **RSA public key auth** — `caching_sha2_password` full auth now works without TLS by requesting the server's RSA public key and encrypting the password with OAEP padding
- **VECTOR type support** — MySQL 9.0+ `MYSQL_TYPE_VECTOR` (0xF2) columns are decoded as raw bytes
- **MySQL 9.x compatibility** — Removed `mysql_native_password` dependency and deprecated `--binlog-format=ROW` option
- **Version-switchable E2E tests** — Docker Compose accepts `MYSQL_VERSION` env var for testing against different MySQL versions

---

## New Features

### 1. RSA Public Key Auth for caching_sha2_password

**Type:** Feature

When `caching_sha2_password` requires full authentication and TLS is not active, the client now requests the server's RSA public key and encrypts the password using RSA-OAEP padding via OpenSSL's EVP API. This eliminates the need for TLS or `mysql_native_password` as a fallback.

**Files Changed:**

- `core/src/protocol/mysql_connection.cpp` — RSA public key request, XOR with scramble, OAEP encryption
- `core/tests/e2e/test_e2e_auth.cpp` — Updated `CachingSha2WithoutTlsColdCache` to expect success
- `e2e/docker/mysql-init/01-init.sql` — All users migrated from `mysql_native_password` to default auth plugin
- `e2e/docker/docker-compose.yml` — Removed `--mysql-native-password=ON` and `--binlog-format=ROW`

### 2. VECTOR Column Type Support (MySQL 9.0+)

**Type:** Feature

Added support for `MYSQL_TYPE_VECTOR` (type code 0xF2), introduced in MySQL 9.0 for storing fixed-dimension float vectors. VECTOR columns use the same wire encoding as BLOB (metadata byte for length prefix size + data) and are exposed as raw bytes through the C ABI.

**Files Changed:**

- `core/src/types.h` — Added `kVector = 0xF2` to `ColumnType` enum
- `core/src/binary_util.cpp` — Added `CalcFieldSize` case for VECTOR using blob pack-length encoding
- `core/src/table_map.cpp` — Added metadata parsing (1-byte pack_len) for VECTOR
- `core/src/row_decoder.cpp` — Added `kVector` to blob decode arm
- `core/src/capi.cpp` — Mapped `kVector` to `MES_COL_BYTES` in C ABI
- `core/tests/e2e/test_e2e_binlog_dml.cpp` — E2E tests for VECTOR insert/update/delete (skipped on MySQL < 9.0)

**Backward compatibility:** No impact on MySQL 8.x replication. Type 0xF2 never appears in 8.x binlog streams.
