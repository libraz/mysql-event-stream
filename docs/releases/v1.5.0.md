# mysql-event-stream v1.5.0 Release Notes

**Release Date:** 2026-07-15
**Type:** Minor (additive API + behavioral change to `mes_client_stop()`)
**Previous Version:** v1.4.0

---

## Overview

Version 1.5.0 gives the high-level client explicit flow control and completes
its introspection surface, then ships prebuilt native artifacts so both bindings
install without a local toolchain.

The C ABI additions are **additive** — no symbols are removed and no struct
layout changes. There is one behavioral change: `mes_client_stop()` now joins
the reader thread before returning (see [Changed](#changed)).

**Highlights:**

- **Byte-budgeted backpressure** — bound the client event queue by total bytes,
  not just event count, so a slow consumer can't grow memory unboundedly on wide
  rows
- **Client-side event-size clamp** — reject oversized binlog events at the reader,
  mirroring the engine cap
- **Streaming-state introspection** — drain a queued terminal error exactly once
  instead of busy-looping on `MES_ERR_DISCONNECTED`
- **Delivery-acknowledged GTID** — `current_gtid` advances only after a committed
  transaction is handed to the consumer
- **Prebuilt distribution** — per-platform Node packages and Python wheels built
  in CI against pinned static dependencies

## New Features

### 1. Byte-budgeted queue backpressure (C ABI + bindings)

`mes_client_set_max_queue_bytes()` / `mes_client_get_max_queue_bytes()` bound the
client event queue by total bytes; `mes_client_queued_bytes()` reports the
current fill. The byte budget complements the existing event-count limit —
whichever limit is reached first applies backpressure. `EventQueue` charges each
entry its packet capacity plus checkpoint storage.

Surfaced as Node `maxQueueBytes` and Python `max_queue_bytes` in the client
config.

### 2. Client-side maximum event size (C ABI + bindings)

`mes_client_set_max_event_size()` / `mes_client_get_max_event_size()` clamp
oversized binlog events at the reader, matching the engine-side cap and its
normalization. Surfaced as Node `maxEventSize` and Python `max_event_size`.

### 3. Streaming-state introspection

`mes_client_is_streaming()` distinguishes a *drainable terminal state* (a queued
terminal error is still available from `mes_client_poll()`) from
`mes_client_is_connected()`. Consumers use it to read the terminal error exactly
once rather than spinning on `MES_ERR_DISCONNECTED`. Surfaced as Node
`isStreaming` and Python `is_streaming`.

### 4. Checksum-state introspection

`mes_client_checksum_enabled()` reports whether the reader detected a CRC32
trailer, so a raw `CdcEngine` consuming the client's poll output can be wired to
the same checksum handling.

### 5. Delivery-acknowledged GTID tracking

A `TransactionGtidTracker` makes `mes_client_current_gtid()` advance only after a
committed transaction has been delivered to the consumer. It is a delivery
acknowledgement, not a durable checkpoint — persist it after your own processing
completes. Surfaced as Node `currentGtid` and Python `current_gtid`.

### 6. Public-key retrieval opt-in

`allow_public_key_retrieval` opts into plaintext `caching_sha2_password`
public-key retrieval for connections without TLS. It is disabled by default and
exposed in both bindings.

### 7. Prebuilt native distribution

The publish workflow builds per-platform Node native packages
(`@libraz/mysql-event-stream-<platform>-<arch>`) and Python wheels against pinned
static dependencies. The Node loader falls back from a local build to the
matching optional package, so consumers install without a compiler.

## Changed

### `mes_client_stop()` is now synchronous

`mes_client_stop()` joins the reader thread before returning. It is therefore no
longer async-signal-safe and must be called from a normal thread, not a signal
handler. This removes the window where the reader could still touch client state
after `stop()` returned.

### Tooling and CI

- `make format` fans out to Biome (Node) and ruff (Python) in addition to
  clang-format.
- Node dependencies and the Python dev-tool version floors were updated to
  current releases.
- Documentation-only changes no longer trigger the build. Sanitizer runs
  (ASan / UBSan / TSan) run on the develop branch and an on-demand safety
  workflow. The cross-server protocol matrix and the fuzz corpus gate the
  publish pipeline.

## Migration Guide

All existing code continues to compile and link — the new C ABI functions are
additive.

**One behavioral change to review:** if you call `mes_client_stop()` from a
signal handler or otherwise rely on it being async-signal-safe, move the call to
a normal thread. `stop()` now blocks until the reader thread has joined.

**Optional adoption:**

- Set `max_queue_bytes` (Node `maxQueueBytes`, Python `max_queue_bytes`) to bound
  client memory on wide-row workloads.
- Set `max_event_size` (Node `maxEventSize`, Python `max_event_size`) to reject
  oversized events at the reader.
- Use `is_streaming` (Node `isStreaming`, Python `is_streaming`) to drain a
  terminal error once instead of polling after disconnect.
- Persist `current_gtid` after your own processing completes; it now reflects
  delivered, committed transactions.
